# discovery plugin

discovery_plugin.lua

# 说明

通过 route.service_name 查找上游服务节点，根据负载规则，转发到上游服务

# 参数

参数名 类型 默认值 说明
discovery_node_not_found_resp string "no server node start" 当没有查询到上游服务节点时返回的数据

# 设置

  1. 添加【路由】 -> 【服务】映射

http://localhost:10000/dashboard/#/gateway/routes

// props
{
  "discovery_node_not_found_resp": "当没有发现服务节点时候返回的消息"
}
  1. 添加服务节点

大多数情况下，服务节点应该是在服务启动时自动注册，控制面板也提供了手动维护功能
